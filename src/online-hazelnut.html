DOM Clobbering Challenge #6

Your goal: Get the site to load a new script named malicious.js, again.

Sometimes the field on the configuration object is nested. Exploiting this requires a new DOM Clobbering technique which involves HTMLCollections. In particular, if you use multiple elements to clobber and those element share a name or id then the clobbered value will be a HTMLCollection. And, you guessed it, a HTMLCollection can itself be clobbered too!

In this challenge, you must use this technique to get the vulnerable script to load a script named malicious.js.

HINT: Which HTML element has a src attribute?

NOTE: This challenge has multiple similar solutions, not all of which work in every browsers! This challenge includes some logs that can help you debug such issues.
